## 🔴 Two live, user-visible bugs (not just style debt)

1. **Invalid `-default` color tokens** (coaching-insights, 6 spots): `bg-green-default`, `text-red-default`, `var(--color-yellow-default)`, etc. The token system has no `-default` shade, so these generate **no color** — heatmap cells, correlation chart bars, and signal/insight icons render with no fill in both light and dark mode. Fails silently (no tsc/lint/runtime error). Files: `heatmap-section.tsx:11`, `correlation-panel.tsx:17-19`, `signals-panel.tsx:16/18`, `insight-line.tsx:12`.
2. **Divergent status pills**: `EvaluationStatusPill` vs `QaStatusPill` color the same `QaEvaluationStatus` enum differently (ai_scored blue vs grey; invalidated grey vs red), and binary Pass is computed `>=1` in one formatter, `===1` in another — so the same ticket can read **Pass on one screen, Fail on another.**

## Cross-cutting themes

- **Enforcement gap is the single root cause.** Conventions live in CLAUDE.md/DESIGN.md but nothing in the build catches violations. The strong dimensions (zero raw hues, zero arbitrary sizes, zero bracket spacing) are clean by discipline alone and will silently regress. Every dimension's systemic fix independently lands on the same answer: a pre-merge lint/grep guard.
- **Sibling-task drift from parallel workers.** Drift clusters in settings/coaching/reports-coaching-insights code shipped by parallel workers who never saw each other's PRs — re-deriving the same value (text-xs group-heading, badge size maps) or worse, *divergent* values (status colors, Pass threshold, `-default` shade). Core surfaces built earlier are clean.
- **Duplicated logic is more dangerous than duplicated markup.** The genuinely user-visible defects all stem from copy-pasted *logic* that diverged (TEAM_TONES map, ratingTone thresholds, Pass/Fail, status colors). Co-locate tone maps + thresholds next to their component, or in `src/lib` where the testing rule forces a co-located test.
- **Primitives should own conventions.** `cursor-pointer` hand-applied 43×, forgotten ~20×; pill spacing re-typed per pill; group-heading re-derived per control. Bake the convention into the base primitive and the drift can't happen.
- **Silent-failure tokens.** Invalid Tailwind utilities + undefined CSS vars produce no error — just absent color. A closed-set token validator catches the whole class at author time.

## Top recommendations

- **P1** — Fix the `-default` token bug (6 instances). Smallest effort, highest user impact; ships broken color today.
- **P1** — Consolidate the two status pills into one shared pill + extract scale-aware score formatting (one Pass threshold) into a tested `src/lib/qa/format-score.ts`.
- **P2** — Add pre-merge lint guards: closed-set token validator, raw-hue ban, `title=` ban on interactive elements, `text-xs` allowlist, raw-element `cursor-` check. Runs in the existing worker pre-flight; no CI runner needed. Would have caught the P1 token bug at author time.
- **P2** — Bake conventions into shared primitives: `cursor-pointer` on the Button base + DrawerLink; extract `GroupHeading`, `CountChip`, `EmptyState`, `SettingsPageHeader`, one badge size-scale constant. Collapses most of the typography + spacing + duplication counts to a handful of source definitions.
- **P3** — Decide the spacing half-step policy once (bless `gap-1.5`/`py-0.5`/`px-2.5` as named tight-chrome tokens, or codemod-snap the ~314 to the nearest allowed step), then enforce via the spacing guard.
- **P3** — File a follow-up to adopt shadcn's sonner toast and retire the hand-rolled `toast.tsx`.
